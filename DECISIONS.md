# Decisions

One line each: what, why.

- Extension lives at repo root (package.json/src/media/test here), not a nested `loopboard/` dir — the workspace this operates on IS this repo, so `workspaceContains:TODO.md` activation and workspace-root file IO line up.
- Tests are plain CommonJS `.js` under `test/`, requiring compiled modules from `out-test/` — keeps devDependencies to `typescript` + `@types/vscode` only (no `@types/node`), per the non-negotiable.
- `tsconfig.test.json` compiles only the pure modules (parser/writer/model/merge) with `types: []` so they never depend on node/vscode typings; the vscode-touching modules are built by the main `tsconfig.json` into `out/`.
- Board carries structural `phaseHeadings` + `sectionExtras` (attached as non-enumerable-ish props via helpers `getPhaseHeadings`/`getSectionExtras`) so exact section headings and HTML-comment templates round-trip verbatim without bloating the documented `Board` interface.
- Parser strips leading `❓`/`⚠️` from question/feedback text and the writer re-adds them — canonicalizes emoji placement so a file missing them normalizes cleanly.
- `make test` runs `node --test 'test/*.test.js'` (glob, not a bare dir) because Node 22 treats a bare `test/` argument as a module path.
- Pure field-patch + conflict logic lives in `merge.ts` (no vscode) so the merge protocol is unit-tested in Docker; `store.ts` will consume it.
- Ported the design bundle's mockup (`loopboard-vscode-extension/project/board.dc.html`, `sidebar.dc.html`) — a React-like DC prototype — into vanilla `media/*.{html,css,js}`: same regions, type scale (15/13/11), 4px grid, colors via `var(--vscode-*)`, and the two allowed animations (attention pulse, refresh flash) with `prefers-reduced-motion` honored.
- Main `tsconfig.json` adds the `DOM` lib (not `@types/node`) to get `setTimeout`/`TextDecoder`/`TextEncoder` in the extension host without violating the "typescript + @types/vscode only" devDependency rule.
- Writing review `feedback:` from the board is a **field patch**, not a phase move — the board only performs the two human gates (New→Backlog, Review→Done); the loop moves a Review task back to In Progress on its next pass per Rule 13. The mockup's optimistic auto-move was prototype-only.
- Pure gate transforms (`gates.ts`) and the pure loop-command builder (`loop.ts`) are split out of `store.ts`/`terminals.ts` (which import vscode) so both are unit-tested in Docker.
- Webview normalizes the model select's `default (opus)` display value to `''` before sending a patch, matching the store's "no model" representation so an unchanged/default model never trips a false same-field conflict.
- Board full-re-renders on each refresh but **defers** applying an incoming board while a field is focused (`pendingBoard`, flushed on `focusout`) so a loop write mid-typing can't discard the user's in-progress input.
- Added `.vscodeignore` so the packaged `.vsix` ships only `out/` + `media/` + manifest/README (26 files) — source, tests, docs, and the design bundle are excluded.
- Board content pane is left-aligned against the rail (`.pane-inner` margin 0, not auto-centered) — on wide windows the mockup's centered column left a huge gap after the rail, making the cards look like a detached second menu.
- Board's left rail removed (it duplicated the sidebar): phases are horizontal tabs in a top bar with the New-story button, cards take the full remaining width; loop controls live only in the sidebar and gained a stop (■) button (`stopLoop` → `TerminalManager.stop()` = dispose without respawn).
- Textareas auto-grow after mount via `requestAnimationFrame` in `render()` — `scrollHeight` is 0 on detached nodes, so growing at build time left multi-line descriptions cropped at 2 rows.
- New tasks are approved with an explicit ✓ Approve button instead of a bare checkbox (same `promote` gate); Review keeps the checkbox + confirm since accepting rewrites two files.
- `loopBoard.autoRecycle` defaults to **off** (user preference): auto-disposing a model's terminal when its task leaves In Progress kills the session mid-flow more often than it helps; ♻ is the manual, deliberate restart.
- Loop start passes the /loop prompt as the claude CLI's initial prompt argument (`claude --model X --permission-mode Y '<prompt>'`, single-quoted with `'\\''` escaping) instead of injecting the prompt text into the running TUI — injecting the multi-line prompt via sendText/sendSequence was unfixably flaky (bracketed paste swallows the newline; Enters within the TUI's paste-detection window get folded into the paste; sendSequence races terminal focus). The arg approach reliably delivers the prompt as argv, but the CLI only *seeds* it into the REPL input (a pasted-text chip) — it does NOT auto-submit. So after launch we send one standalone Enter (`terminal.sendText('', true)`) after `SUBMIT_DELAY_MS` (3.5s), once the CLI has booted and its paste-detection window has closed; a lone Enter after that window is not folded into the paste. `loopBoard.startupDelayMs` stays removed (the delay is now a fixed internal constant, not the old inject-the-prompt delay); ♻ restart re-arms the loop automatically.
- Fresh workspaces get a "Create TODO.md & DONE.md" button (sidebar + board empty state): the initial tracker ships as `media/todo-template.md` (test-pinned: parses empty, fixpoint, `{MODEL}` block present); `Store.createInitialFiles` never overwrites existing files. Works pre-TODO.md because contributed views activate the extension implicitly (`onView`).
- Renamed the extension "Claude TODO Board" → **LoopBoard** (t-8823): display name "LoopBoard", package/publisher `loopboard`, id prefix `loopBoard.*` (keeps the camelCase convention). Clean break on `claudeTodo.*` settings keys (no aliases — unpublished at 0.1.x, README migration note instead). Terminal names stay "Claude <Model>" (they host the actual claude CLI). Stale `claude-todo-board-vscode-extension/` duplicate deleted.
- Release pipeline (t-9c4e) ports ClockClock's semantic-release setup: `.releaserc.json` with the commit-analyzer/release-notes-generator/github plugin trio, `release.yml` with a `release` job (cycjimmy/semantic-release-action@v4, GITHUB_TOKEN) and a gated `vsix` job that builds (npm ci + tsc + node --test on Node 22 — CI runners aren't the host, so the Docker-only rule doesn't apply) and packages with `vsce package <version> --no-dependencies --no-git-tag-version --no-update-package-json` — version is injected into the vsix manifest only; package.json's version stays frozen in git (no bot bump commits). Artifact goes to the GitHub Release via `gh release upload`; Marketplace publish deliberately deferred (t-a37f follow-up). `environment: All` omitted (ClockClock-specific). `.github/**` + `.releaserc.json` added to `.vscodeignore` (they leaked into the vsix, 30→32 files).
- Loop terminals get a tiny bootstrap paste, not the full prompt (t-1d91): `buildLoopCommand` now emits `/loop <interval> You are running as model <model>. Open TODO.md, read the loop worker instructions in its ## Automation section, and follow them exactly...` — the Automation fenced block holds the standing model-agnostic instructions, so editing TODO.md retunes running loops on their next pass (only the interval is fixed at spawn). Chosen over startup validation (shell-integration exit-code checks only catch fast failures; tmux stays the v2 path — user decision: no follow-up story) and over the one-command argv variant (still rejected: `/loop` as claude argv is sent as literal text).
